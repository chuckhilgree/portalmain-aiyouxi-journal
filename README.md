# portalmain-aiyouxi-journal

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存放，不针对任何特定域名或网站，仅作为页面内容的结构化存档。

## 项目简介

该项目是一个静态 HTML 页面集合仓库，主要功能包括：

- 归档多个独立页面，保留其原始结构与样式
- 支持通过 GitHub Pages 等方式进行页面浏览
- 保持文件结构清晰，便于后续维护与扩展

## 目录说明

```
├── index.html          # 主入口页面（可选）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 公共资源文件（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：所有独立 HTML 页面存放于此，每个文件对应一个独立页面。
- `assets/`：存放页面所需的公共样式、脚本与图片资源。

## 页面归档说明

- 每个 HTML 页面均为独立文件，可直接通过浏览器打开。
- 页面之间无强依赖关系，可单独访问或修改。
- 归档内容不针对任何特定域名或网站，仅作为静态页面集合。

## 维护说明

- 如需新增页面，请在 `pages/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。
- 如有公共资源更新，请修改 `assets/` 下对应文件。
- 建议保持文件命名简洁，避免特殊字符。
- 本仓库不涉及任何动态内容或后端服务，仅作为静态文件托管。
